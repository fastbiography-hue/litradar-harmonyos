# 风险登记表

## 已识别风险

### R-01: PubMed API 可达性
- **状态**: 持续监控
- **影响**: FetchEngine 返回 0 篇文献
- **根因**: 国际网络链路不稳定
- **缓解**: 超时 60s + 返回空数组 + console.error 日志
- **用户感知**: Toast "抓取失败"

### R-02: ArkTS 严格模式限制
- **状态**: 已解决
- **影响**: 编译错误
- **已知限制**: 无解构赋值、无括号索引、无 `throw e` 裸抛、必须显式返回类型
- **缓解**: 写入 `docs/harmonyos-rules.md`，所有代码遵循

### R-03: WorkScheduler 行为不确定
- **状态**: 待验证
- **影响**: 后台定时任务可能不触发或延迟
- **根因**: 鸿蒙PC后台调度策略未文档化
- **缓解**: 保留"立即抓取"手动触发，不依赖定时任务

### R-04: Jaccard vs difflib 去重偏差
- **状态**: 已接受
- **影响**: 边缘标题去重结果与 Python 不一致
- **根因**: ArkTS 无 difflib，用 Jaccard 0.85 近似
- **缓解**: 大部分实际论文标题场景下偏差可忽略

### R-05: AI 供应商 API 差异
- **状态**: M3 待处理
- **影响**: DeepSeek/OpenAI/豆包/通义 接口格式不同
- **缓解**: AiService 统一接口 + provider adapter 模式

### R-06: Zotero API 认证复杂度
- **状态**: M3 待处理
- **影响**: 双向同步可能需要 OAuth
- **缓解**: P2 优先级，先实现单向导入

### R-07: 鸿蒙PC 设备类型 API 兼容性
- **状态**: 已接受
- **影响**: 部分 API 在 2in1 设备上行为可能与文档不一致
- **缓解**: 使用 `deviceTypes: ["phone","tablet","2in1"]` 最大兼容
