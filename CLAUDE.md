# CLAUDE.md — LitRadar HarmonyOS

## 核心指令

每次开始任务前，你必须按顺序读取以下文件以建立上下文：

1. `README.md` — 项目概述、架构、版本
2. `ARCHITECTURE.md` — 模块边界、分层规则
3. `ROADMAP.md` — 长期路线图
4. `TASKS.md` — 当前可执行任务列表

## 工作规则

### 一次只做一个任务
- 从 `TASKS.md` 中选择一个状态为 `pending` 的任务
- 将该任务状态改为 `in_progress`
- 完成任务后改为 `completed`，然后停止

### 只能做 TASKS.md 中列出的任务
- 不允许自行发明新任务
- 不允许"顺手"修改无关代码
- 如果发现需要新任务，记录在 TASKS.md 末尾但不执行

### 不允许
- 大规模重构（一次改 3 个以上文件必须说明理由）
- 修改与任务无关的文件
- 新增运行时第三方依赖（`@ohos.*` 系统 API 除外）
- 把密钥、密码、证书写进仓库
- 修改 `backend/` 目录下的任何文件（它只作为旧逻辑参考）

### 完成后必须输出
```
修改文件: xxx.ets, yyy.ets
验证方式: <如何确认修改有效>
风险: <无 / 有，具体说明>
```

## 项目约束

- 纯 ArkTS 原生，零 Python 后端依赖
- 只使用 `@ohos.*` 系统 API
- `papers` 表 schema 不可变（22 字段）
- `backend/` 目录是历史遗产，只读参考
